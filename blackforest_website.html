Tender Search & Scraping Utility for Multi-Portal Government Procurement Data

Cloud84 Black Forest Project is a desktop utility built for reliable tender collection across multiple portals, with GUI + CLI modes, resilient scraping workflows, and centralized SQLite persistence for audit-ready historical search.

Current Version: 2.1.9

What This Tool Does

End-to-end tender discovery, extraction, persistence, and export with operational controls.

Department Scraping

Scrapes tenders by organization/department from supported procurement portals.

Batch Multi-Portal Runs

Runs multiple portals with dashboard visibility, retries, and structured run reporting.

Refresh Watch Automation

Detects portal changes and triggers scraping only when updates are found.

Central SQLite Store

Persists run metadata and tender rows as source-of-truth, then exports user files from DB views.

Data Integrity Rules

Keeps latest row per portal+tender ID and removes missing/invalid tender IDs during persistence.

Excel/CSV Outputs

Produces operator-friendly output files while preserving traceability with run and DB references.

Key Innovations

Practical reliability and long-term data architecture improvements introduced in recent releases.

Adaptive Portal Recovery

Locator memory, retry orchestration, and navigation fallbacks reduce portal-break failures.

Delta & Resume Intelligence

Manifest-backed "only new" behavior with optional second-pass sweep captures late-arriving tenders.

Tiered Backup Strategy

Automatic daily, weekly, monthly, and yearly SQLite snapshots with retention windows.

Scalable Historical Imports

High-volume import workflow with batched de-duplication and normalized lookup indexing.
